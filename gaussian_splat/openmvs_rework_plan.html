Gaussian Splat 回修計畫

參考 OpenMVS 後段的貼圖 QA、遮罩、縮圖、重轉與檢視器收尾，把 3DGS 從「只有 splat 成品」推進到「可診斷、可比較、可回修」的工作流。

回 Gaussian Splat
核心判斷

OpenMVS 的 TextureMesh 是 UV mesh 貼圖流程，不能直接貼到 3DGS splat 上。3DGS 能直接借用的是 QA、遮罩、viewer cache、縮圖和重轉防呆；若要有貼皮模型，應該產出 proxy mesh/GLB 或雙軌 OpenMVS textured GLB。

A. 先做

QA / Viewer 收尾

最像 OpenMVS 後段，風險低，也能馬上量化失敗原因。

splat.ply / splat.clean.ply
↓
黑色比例、白色/透明比例、低 opacity、bbox、點數
↓
列表 QA、cache bust、縮圖、重轉耗時重算
B. 接著做

3DGS → Mesh 預覽

把 splat 轉 colored point cloud，再轉 proxy mesh/GLB；這是輔助成果，不是原生 UV 貼圖。

splat.clean.ply
↓
colored point cloud / filter report
↓
cleaned_mesh.glb + mesh color QA + compare viewer
C. 有需要再做

3DGS + OpenMVS 雙軌

同一批影像同時產出 splat 與 textured GLB，品質最穩，但成本最高。

frames / masks
↓
3DGS splat + OpenMVS textured GLB
↓
並排 viewer + QA gate + 交付模式判斷

狀態與防呆按鈕

待處理
暫停
轉檔中
已完成
失敗
  • 失敗、已完成：可重轉，重轉前清除舊輸出與舊 QA。
  • 暫停：可啟動。
  • 轉檔中：可暫停，透過 .abort 中止 worker。
  • 待處理、暫停、已完成：可刪除，設定 del=1。

Task 與檢核項

1 優先

管理端狀態一致化

補齊待處理、暫停、轉檔中、已完成、失敗，並把 OpenMVS 的防呆按鈕行為搬回 3DGS。

  • 檢核：每個狀態只顯示合法按鈕。
  • 檢核：重轉耗時從本次 work_st_datetime 算起。
2 優先

3DGS QA 擴充

在 qa_report.json 補 splat 黑色比例、白/透明比例、低 opacity、scale outlier、clean 保留率。

  • 檢核：黑烏烏時能量化，不只顯示失敗。
  • 檢核：列表能顯示主要 QA 摘要。
3 優先

Viewer cache 與縮圖

參考 OpenMVS：完成後產生縮圖，viewer 對 splat/GLB/sidecar 加 cache bust。

  • 檢核：重轉後同 URL 不會看到舊成果。
  • 檢核：列表縮圖會跟著更新。
4 第二階段

Proxy mesh / GLB 強化

沿用 splat_to_pointcloud.py、pointcloud_to_mesh.py、build_compare_bundle.py，補 mesh color QA 與 compare report。

  • 檢核：完成 job 可產出 compare/mesh/cleaned_mesh.glb。
  • 檢核：compare viewer 可左右比較 splat 與 mesh。
5 第二階段

遮罩 / 主體隔離

讓 3DGS 支援 ZIP 內 .mask.png 或 ROI mask，在訓練前就排除動態背景與天空。

  • 檢核：QA 記錄 mask count 與 mask mode。
  • 檢核：無 mask、有 mask 的 job 可並排比較。
6 實驗

雙軌 OpenMVS textured GLB

同一批影格同時跑 3DGS 與 OpenMVS，讓 splat 負責真實感，OpenMVS 負責可貼圖 mesh。

  • 檢核：同 job 能看到 splat 與 textured GLB。
  • 檢核：報告能建議交付模式。

第一輪驗收

資料表

新欄位 migration 可重跑，既有 job 不壞。

轉檔

重轉、暫停、啟動、刪除都能正確影響排程。

QA

qa_report.json 能說明視覺品質問題。

展示

列表、viewer、縮圖、compare viewer 可從 3wa 網址直接看。
